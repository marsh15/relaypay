# RelayPay v0.3.0 — Immutable reconciliation evidence

RelayPay v0.3.0 adds bounded provider-statement ingestion, PostgreSQL-leased reconciliation, and
an operator mismatch workflow while preserving the v0.1 lost-response contract and `/api/v1`
compatibility.

> **Synthetic data only.** This release is not a payment processor and must never receive real
> card, bank-account, identity, customer, statement, or other regulated data. It is not deployed
> or hosted.

## Added in this release

- Immutable mock-provider JSON statement exports with exact bytes, SHA-256 digest, stable key,
  provider item ID, operation kind, amount, currency, status, and occurrence time.
- Bounded CSV/JSON imports scoped to one organisation environment. Identical source and bytes
  return the existing import and run; changed bytes return `STATEMENT_SOURCE_CONFLICT`.
- Versioned reconciliation runs claimed with PostgreSQL `FOR UPDATE SKIP LOCKED`, opaque lease
  tokens, expiry, and stale-token rejection.
- Deterministic exact matches and evidence for missing internal transactions, missing provider
  transactions, amount, currency, and status differences, duplicate provider effects, and
  successful capture/refund evidence without a journal.
- Immutable mismatch evidence versions and append-only workflow history around the validated
  `OPEN → ACKNOWLEDGED → RESOLVED` transition.
- Organisation-admin statement and workflow commands protected by session authentication and
  CSRF. Read paths remain tenant and environment scoped.
- Celery and PostgreSQL-poller reconciliation workers with no network I/O inside transactions.

## Compatibility and migration

M2 creates new evidence and workflow tables; it does not backfill or rewrite v0.2 payment,
provider, ledger, event, delivery, identity, or environment rows. Provider statement tables also
start empty. CI upgrades a representative v0.2 fixture in disposable schemas and verifies every
pre-existing table digest is unchanged.

Read and follow the [v0.3.0 migration guide](../migrations/v0.3.0.md). The permanent v0.1 upgrade,
lost-response, empty-database, drift, backend, console, Compose, Playwright, and axe gates remain
part of release qualification.

## Runnable synthetic proof

With the complete local stack running:

```bash
RELAYPAY_DEMO_PASSWORD='RelayPay-Northstar-2026!' make reconciliation-demo
```

The command exports exact mock-provider bytes, imports them into the seeded TEST environment,
runs the database reconciliation worker, and prints only synthetic identifiers, digest, and
counts.

## Known limitations

- Internal payment and ledger authority remains INR-only. Imported three-letter currency codes
  can differ only so reconciliation can preserve and report provider evidence.
- Statement input is limited to 1 MiB and 10,000 items and supports the bundled synthetic payment
  provider contract only.
- Mismatch workflow is API-first; the environment-aware console arrives in M8.
- Resolution cannot create compensation, change outcomes, or mutate old evidence. It can only
  record notes, refresh evidence, and link an existing compensation journal.
- No production SLO, PCI claim, real-money handling, deployment, hosting, or owner-account work.
