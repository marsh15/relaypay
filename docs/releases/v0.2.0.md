# RelayPay v0.2.0 — Identity and environment isolation

RelayPay v0.2.0 makes identity global and every tenant workflow environment-aware while retaining
the v0.1 lost-response safety contract and `/api/v1` compatibility.

> **Synthetic data only.** This release is not a payment processor and must never receive real
> card, bank-account, identity, customer, or other regulated data. It is not deployed or hosted.

## Added in this release

- Global users with organisation memberships and `ORGANISATION_ADMIN`, `DEVELOPER`, and `VIEWER`
  roles.
- One isolated TEST and one LIVE_LIKE environment for every organisation.
- Environment-bound, versioned API keys with one-time secret display, pending rotation,
  activation, scope changes, revocation, and environment-specific prefixes.
- Idempotent platform-administrator bootstrap and platform-only organisation provisioning.
- Append-only transactional audit records for membership, organisation, and API-key mutations.
- Composite PostgreSQL constraints that reject cross-environment resource links and let business
  keys be reused independently in TEST and LIVE_LIKE.

## Compatibility and migration

The existing `/api/v1` request/response contract, idempotent replay bytes, and lost-response proof
remain compatible. Existing v0.1 rows backfill to TEST, LIVE_LIKE starts empty, migrated keys
become active version 1 keys, and immutable evidence remains byte-for-byte unchanged.

Read and follow the [v0.2.0 migration guide](../migrations/v0.2.0.md). CI runs the same deterministic
upgrade fixture before the empty-database correctness, console, Compose, Playwright, axe, and
lost-response gates.

## Known limitations

- INR-only synthetic data and deterministic local mocks.
- No self-service user invitation flow; membership assignment targets an existing global user.
- The console remains the v0.1 evidence surface; the environment-aware console expansion is M8.
- Reconciliation, settlement, payouts, connectors, SDKs, observability expansion, and edge proof
  remain scheduled for later milestones.
- No production SLO, PCI claim, real-money handling, deployment, or hosting.
